# Layout

Layout expresses institutional structure.

Every position must communicate a relationship.

## Structure

Organize information according to its institutional role.

Do not organize interfaces according to implementation details.

## Reading flow

Support a continuous reading experience.

Interfaces should naturally guide the eye from understanding to inspection.

Avoid abrupt visual interruptions.

## Alignment

Alignment establishes relationships.

Elements sharing the same role should share the same alignment.

Misalignment should communicate a deliberate distinction.

## Space

Whitespace is structural.

Use spacing to separate concepts, not to decorate layouts.

Increase spacing before introducing additional separators.

## Grids

Grids provide consistency.

Break the grid only when the institutional meaning requires it.

## Columns

Columns represent distinct semantic roles.

Do not use multiple columns solely to increase information density.

## Sections

Each section should represent a coherent institutional concern.

Avoid mixing unrelated concepts within the same section.

## Responsiveness

Responsive layouts preserve meaning.

Content may be reorganized across screen sizes.

Institutional hierarchy must remain unchanged.
At a 320 CSS-pixel viewport, reflow or stack columns so all content and controls remain available without two-dimensional scrolling, horizontal clipping, or loss of function, while preserving the same institutional hierarchy.

## Stability

Stable information should remain visually stable.

Avoid unnecessary layout shifts.

New institutional facts should integrate naturally into the existing structure.
