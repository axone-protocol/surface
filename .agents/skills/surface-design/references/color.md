# Color

Color communicates institutional state.

Color is semantic.

It must never be applied solely for aesthetic or branding purposes.

## Color follows protocol

Color reflects protocol state.

It never represents opinion, emotion or emphasis.

Institutional facts remain readable without relying on color alone.
Every semantic state—including active, warning, error, unavailable, focus, recorded, and revoked—MUST also have a concurrent non-color cue: visible text, an icon or shape with an accessible name, or both.

## Principles

Every color must express meaning.

Equivalent states must use equivalent colors.

Different states must remain visually distinguishable.

## State

Use color to communicate state.

Examples include:

- active;
- recorded;
- revoked;
- warning;
- error;
- unavailable.

## Hierarchy

Hierarchy should rely primarily on typography and layout.

Use color to reinforce hierarchy, not to create it.

## Permanence

Stable institutional facts should use stable colors.

Transient protocol activity may temporarily introduce accent colors.

## Neutrality

Large interface surfaces should remain visually neutral.

Neutral backgrounds improve the readability of institutional assertions.

## Contrast

Maintain WCAG 2.2 AA contrast: at least 4.5:1 for normal text, 3:1 for large text, and 3:1 for non-text controls, focus indicators, and state graphics against adjacent colors.
Color must never reduce accessibility.

## Interaction

Interactive elements communicate affordance through state changes.

Hover, focus, active and disabled states must remain consistent across Surface.
A persistent keyboard focus indicator MUST be visually distinguishable and not conveyed by color alone.

## Status

Prefer a small, consistent palette of semantic colors.

Avoid introducing new colors for individual components.

## Consistency

A semantic color always carries the same meaning throughout Surface.
