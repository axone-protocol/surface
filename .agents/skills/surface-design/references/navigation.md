# Navigation

Navigation follows institutional relationships.

Interfaces should allow users to move naturally between observable protocol facts.

## Navigation follows protocol

Navigation exposes relationships established by the protocol.

Do not invent navigation paths unsupported by protocol facts.

## Principles

Every destination should answer a question raised by the current view.

Navigation should reveal context, not hide it.

Users should always understand why a destination is reachable.
Navigation destinations MUST use semantic links or controls with accessible names, be keyboard-operable, and expose a visible focus indicator that is not conveyed by color alone. A persistent navigation set MUST identify its current destination.

## Navigation follows assertions

Users primarily navigate through institutional assertions.

Technical evidence supports navigation but does not drive it.

## Navigation starts from facts

Navigation begins from observable protocol facts.

Examples include:

- an identity;
- a governance;
- a credential;
- a decision;
- a resource.

## Relationships

Navigation follows explicit relationships.

Examples include:

- issued by;
- issued to;
- governed by;
- decided by;
- established on.

Future derived relationships may be introduced when explicitly established.

## Progressive exploration

Primary interfaces expose the most relevant relationships.

Inspection progressively reveals additional connections.

## Stable destinations

Institutional facts should have stable destinations.

Stable references allow users to revisit, inspect and share institutional facts.

## Context preservation

Navigation should preserve context whenever possible.

Users should understand where they came from and why they reached the current view.

Avoid disruptive navigation patterns.

## Search

Search is a form of navigation.

Search results should expose institutional facts before technical details.

Results should naturally lead to further exploration.
Search results MUST meet the same semantic, accessible-name, keyboard-operability, focus-indicator, and current-destination requirements where applicable.
